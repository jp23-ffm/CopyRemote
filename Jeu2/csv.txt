Format CSV import

  4 colonnes — format historique, inchangé :
  hostname;type;servicenow;notes
  server1;INFO;RITM123456;In ME until 24th
  server2;WARNING;INC123456;OPENSVC Cluster issue

  5 colonnes — avec colonne action :
  hostname;type;servicenow;notes;action
  server1;INFO;RITM999;New issue detected;add
  server2;;;;resolve

  - action=add (ou vide) : ajoute une nouvelle annotation active
  - action=resolve : clôt la dernière annotation active du serveur (les colonnes type/servicenow/notes sont ignorées
  pour cette ligne)
  - Colonne absente = comportement add par défaut → aucune migration nécessaire pour les scripts existants
